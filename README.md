# Assignment_Dashboard

一个面向班委、学委、课代表和老师的 Windows 本地作业分类与提交追踪工具。

Assignment_Dashboard 主要用于监控微信接收的作业文件，同时允许用户自由添加其他本地监听目录。系统会根据学生姓名、课程规则和作业批次整理提交文件，并在本地网页中展示提交进度、未提交名单、待归档文件和最近记录。

当前版本：`v0.1.0`

## 适合谁使用

- 需要收集班级作业的班委、学委和课代表
- 需要核对提交名单和缺交情况的老师或助教
- 希望把微信文件整理、作业归档和进度统计集中到一个界面的使用者

## 主要功能

- 自动监控微信接收文件夹，以及用户设置的其他本地文件目录
- 根据学生姓名、课程关键词和作业批次识别提交文件
- 展示班级总人数、已提交人数、未提交名单和完成率
- 按课程查看作业进度、截止提醒和最近收到的文件
- 对识别不完整的文件进入待归档队列，不会直接错误归档
- 支持手动修改课程、确认具体作业和重新分类
- 支持学生名单、课程、作业批次和扫描目录管理
- 支持规则别名、课程关键词和人工修正记忆
- 支持 PDF、图片和 Word 文档预览
- 支持作业文件打包、下载和公示文件夹同步
- 提供经典版和现代版两套界面
- 提供多种主题、低功耗模式和减少动效适配
- 支持本机访问以及带口令的局域网访问
- 支持公告展示、更新包加载、更新前备份和失败回滚

## 分类方式

系统采用“先识别课程，再确认具体作业”的两层分类方式：

1. 从文件名中识别学生和课程信息。
2. 在对应课程中查找可能的作业或实验批次。
3. 只有信息足够明确时才自动归档。
4. 课程冲突、作业不明确或信息不足的文件会保留在待归档或未归类队列中。
5. 用户确认后的课程别名和修正记录可用于后续同类文件。

这种方式优先避免误分类。文件名越规范，自动识别效果越稳定。

推荐文件名示例：

```text
张三_数字电子技术_实验1.docx
李四_Python_课程作业2.pdf
王五_课程设计_最终版.zip
```

## 分类大脑

现代版界面提供“分类大脑”，用于管理本学期的课程识别规则：

- 设置本学期正式课程
- 维护课程别名和关键词
- 确认或拒绝待确认别名
- 导入、导出和合并专业规则包
- 查看科目修正记忆和非本作业反馈
- 使用文件名样本辅助整理新课程关键词

新科目训练向导只处理用户选择文件的文件名，不读取或上传文件正文。未知课程和冲突结果不会直接自动归档。

## 下载与安装

推荐从项目的 [Releases](https://github.com/Trip1eY/Assignment_Dashboard/releases) 页面下载最新安装程序。

安装程序通常命名为：

```text
作业追踪器v0.1.0.exe
```

运行安装向导后，根据提示选择安装目录、班级名称和作业目录即可。

> 安装程序暂未进行商业代码签名，Windows 可能显示“未知发布者”或 SmartScreen 提示。请只从本项目 GitHub Releases 页面下载。

## 从源码运行

需要自行运行源码时，请先安装 Python 3.10 或更高版本，推荐 Python 3.12。

```batch
git clone https://github.com/Trip1eY/Assignment_Dashboard.git
cd Assignment_Dashboard
python server.py
```

文档预览相关可选依赖：

```batch
pip install python-docx PyPDF2 pywin32
```

缺少可选依赖不会影响基础统计和文件分类，但部分 Word、PDF 文本解析或预览功能可能降级。


可用界面：

```text
http://localhost:18765/dashboard   经典版
```

## 首次使用

现代版帮助页提供“初始化指导”。建议按以下顺序完成设置：

1. 添加或批量导入学生名单。
2. 设置需要监听的目录，例如微信接收文件夹、课程文件夹或其他作业收集目录。
3. 设置班级作业归档目录和公示文件夹。
4. 配置本学期课程名称、别名和关键词。
5. 创建需要统计的作业或实验批次。
6. 执行首次扫描并处理待归档、未归类文件。

初始化进度会根据当前配置和真实运行状态自动更新。

## 公示文件夹

公示文件夹是面向课程查看、分享和历史恢复的文件副本目录，不是系统内部的主要归档目录，也不是实时监听目录。系统内部归档目录通常称为“已收作业”。

文件被完整识别出学生、科目和具体作业后，会先归档到“已收作业”，再同步一份到公示文件夹。公示文件夹通常按以下结构保存文件：

```text
公示文件夹/
  科目/
    作业批次/
      学生姓名_文件名.docx
```

公示文件夹主要用于：

- 给老师或班级集中查看已经确认归档的作业。
- 换电脑、重装系统或提交记录不完整时，回填历史作业。
- 手动整理历史文件后，让系统重新识别并归档。

公示文件夹不会参与实时 watcher，避免系统复制出来的副本被再次当成新文件。手动放入文件后，需要点击“从公示目录回填已收作业”；服务器启动时，在公示文件夹已启用的情况下也会自动执行一次回填。回填会尝试根据文件名和目录结构匹配学生、科目与作业，并处理重复学生名前缀。

只有学生、科目和具体作业都确认后，文件才会同步到公示文件夹。只识别出科目但作业待确认的文件，会留在待归档队列中。

关闭公示文件夹后，系统会停止双写、启动回填和手动回填；按确认删除公示文件夹时，不会影响“已收作业”中的内部归档文件。

## 待归档与未归类文件

### 待归档文件

系统已经识别出课程，但还不能确定具体作业时，文件会进入待归档队列。确认具体作业后才会归档和同步到公示文件夹。

### 未归类文件

系统无法可靠识别学生或课程时，文件会进入未归类队列。可以预览文件、重新判断或手动指定信息。

微信接收文件夹是主要使用场景，但不是唯一支持的来源。用户可以添加其他本地监听目录；系统不会仅因为文件位于某个监听目录中，就直接猜测学生或具体作业。

## 文件预览

- PDF 和图片使用浏览器直接预览。
- DOCX/DOC 文件首次预览时会进入后台转换队列。
- 安装 Microsoft Word 时优先使用 Word 转换为 PDF。
- 没有 Word 时会尝试使用 LibreOffice。
- 转换工具不可用时会降级为文本预览，并保留打开原文件的入口。
- 已转换文档会使用本地缓存，提高再次预览速度。

在管理页的“文件预览预热”中，可以开启“启动时自动预热”，也可以随时点击“立即预热”。预热优先处理最近 20 个仪表盘 Word 文件，并从已收作业和已启用的公示文件夹补足，不遍历全部微信历史。Word 转换运行在隐藏隔离进程中，宏、提醒和最近文件记录会被关闭；单个异常文件超过时限后会终止本次转换，不会阻塞服务器或自动打开原文件。

## 本机与局域网访问

系统默认只允许本机访问，其他设备无法直接连接。

需要在手机或另一台电脑查看时，可以在管理页开启“允许局域网访问”。开启后系统会：

- 显示局域网访问地址
- 生成随机访问口令
- 要求远程设备验证后才能进入
- 重启服务以应用访问模式

访问模式和口令只能在运行服务的电脑上修改。

> 局域网访问只适用于可信私人网络。请勿通过路由器端口映射、内网穿透或云服务器将本项目直接暴露到公网。

## 系统更新

进入管理页的“系统更新”模块，可以：

- 检查项目发布页中的新版本
- 下载并加载最新更新包
- 手动选择本地 ZIP 更新包

更新前会自动备份当前程序和运行数据。更新失败时会尝试自动回滚。

无法访问 GitHub 时，可以从 Releases 页面手动下载更新包，再在系统更新模块中加载。

## 数据与隐私

学生名单、提交记录、课程规则、扫描目录和预览缓存默认保存在运行程序的电脑上。项目不会主动把这些数据上传到 GitHub。

使用时请注意：

- 不要把真实学生名单、作业文件或访问口令发到公开 Issue 或 Discussion。
- 截图反馈问题前，请遮挡学生姓名、学号和本机完整路径。
- 将规则生成提示词交给外部 AI 前，请确认其中不包含学生信息、访问口令或私人路径。
- 备份、复制或迁移安装目录时，请妥善保管其中的运行数据。

## 常见问题

### 启动后网页打不开

先尝试打开 `http://localhost:18765`。如果提示端口已占用，可能已经有一个服务正在运行；关闭旧启动窗口后再重新启动。

### 监听目录中出现文件，但仪表盘没有显示

检查该目录是否已经加入扫描设置，并确认文件已实际保存到本机。使用微信接收文件时，还需要确认微信已经完成下载；微信未开启自动下载时，文件不会出现在本机目录中。

### 文件没有自动归档

检查文件是否进入待归档或未归类队列。文件名缺少学生、课程或作业信息时，需要手动确认。

### Word 文档无法预览

确认电脑已安装 Microsoft Word、LibreOffice 或 `python-docx`。即使无法转换预览，仍可以打开原文件。

如果管理页显示预热失败，可以点击“立即预热”重试并查看失败数量。受密码保护、严重损坏或需要人工确认修复的 Word 文件会被安全跳过，不会弹出交互窗口；这类文件需要手动打开并另存为正常的 `.docx` 后再预览。

### 手机无法访问

确认管理页已开启局域网访问，手机与电脑连接同一可信网络，并使用页面显示的地址和访问口令。

### 更新失败怎么办

系统会在更新前创建备份并尝试自动回滚。如果服务没有自动恢复，可以关闭启动窗口后重新运行“启动作业追踪器.bat”，或使用安装目录中的更新修复工具。
